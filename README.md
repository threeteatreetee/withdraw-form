# ใบเบิกเงิน — เว็บสร้างเอกสารเบิกเงิน (ส.การโยธา 1993)

กรอกใบ **"ขอเบิกเงินค่าแรงล่วงหน้าหรือค่าวัสดุ"** บนจอ → กดปรินต์ (Ctrl/⌘+P) ได้ PDF/กระดาษหน้าตาเหมือนฟอร์มเดิมไปเซ็น.
เก็บ "รายชื่อคน + บัญชี" ไว้เลือกซ้ำได้ (เลือกคน → เติมชื่อ/ธนาคาร/เลขบัญชีให้อัตโนมัติ).

## ไฟล์
| ไฟล์ | หน้าที่ |
|---|---|
| `index.html` | หน้าเว็บจริง (login ก่อนใช้) — 2 หน้า: สร้างใบเบิก / รายชื่อบัญชี |
| `preview.html` | **พรีวิว template แบบไม่ต้อง login** (ข้อมูลตัวอย่าง) — ไว้ดู/ปรับหน้าตาเอกสาร |

| `style.css` | สไตล์ทั้งหมด (จอ + ปรินต์) |
| `schema.sql` | สร้างตาราง `accounts` + RLS |

## ตั้งค่า Supabase (ครั้งเดียว)
ใช้ **โปรเจกต์เดียวกับ task-tracker** (`wocqftaojrviqjprzrjw`) — ค่า URL/anon key ใส่ไว้แล้วใน `app.js`.
1. Supabase → **SQL Editor** → วาง `schema.sql` ทั้งก้อน → **Run**
2. Supabase → **Authentication → Users → Add user** → สร้างอีเมล/รหัสไว้ล็อกอิน (แชร์เฉพาะคนในทีม)
3. เปิด `index.html` → ล็อกอินด้วยอีเมล/รหัสข้อ 2

> เลขบัญชีคนจริง → RLS ล็อกให้เฉพาะคน login เท่านั้นอ่าน/เขียนได้. คนมี URL แต่ไม่ login → เข้าไม่ถึงข้อมูล.

## การคำนวณ (ในไฟล์ template)
```
รวมรายการ = Σ แถวค่าแรง/วัสดุ
คงเหลือ   = รวมรายการ − เบิกล่วงหน้า
ภาษี      = ภาษี% × คงเหลือ        (default 3%, แก้ได้)
ประกัน    = ประกัน% × รวมรายการ    (default 5%, แก้ได้)
คงเหลือเป็นเงิน = คงเหลือ − ภาษี − ประกัน
```
ใบไหนไม่หัก → ตั้ง % = 0. (ตรวจแล้วตรงกับใบตัวอย่าง 38,614.76)

## เพิ่ม template เอกสารใหม่ (อนาคต)
1. ก๊อป `templates/bik.js` → `templates/xxx.js`
2. แก้ `render()` (วาดฟอร์ม) + `compute()` (สูตร) แล้วเปลี่ยน `id`/`title`
3. เพิ่ม `<script src="templates/xxx.js"></script>` ใน `index.html` (ก่อน `app.js`)
4. มี > 1 template → dropdown เลือกจะโผล่บน topbar ให้เอง

ไม่ต้องแตะ `app.js` (shell รู้จัก template ผ่าน `window.TEMPLATES`).
